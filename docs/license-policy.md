## Core rules

The repository uses a restrictive regime with three cumulative requirements for
any use in another project:

1. visible credit to the author or rights holder;
2. prior notification to the rights holder;
3. prior written authorization from the rights holder.

Credit and notification are mandatory, but they are not enough by themselves.
They never replace prior written authorization.

## Required notification path

When notification is required, it must be sent through both of these channels:

- at least one contact method published, linked, or otherwise related to the
  rights holder's GitHub presence associated with this repository;
- email to `naml@electrondevnaml.xyz`.

If only one channel is used, the notification requirement is not fully met.

## What counts as visible credit

Visible credit must be clear and reasonably prominent. Depending on the form of
use, it should appear in a place such as:

- a `README.md` or repository landing page;
- a credits or acknowledgements section;
- an about page or application credits screen;
- end-user or developer documentation.

At minimum, the credit should identify the original repository or project and
the author or rights holder.

- imply that the repository is open source or generally reusable.

## Private evaluation only

Absent separate written permission, the safe default is private evaluation,
review, testing, and compatibility analysis only.

If the intended activity involves another project in any way, assume it is not
authorized until the rights holder has been notified through the required
channels and has granted prior written authorization.

## Practical reminder

Before reusing anything from DXBridge outside this repository, do all of the
following first:

1. prepare visible credit;
